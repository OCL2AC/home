OCL2AC Tool

   OCL2AC is a tool based on the Eclipse Modeling Framework (EMF) and consists of two main components:
    (1) OCL2NGC translates OCL constraints into nested graph constraints.
    (2) NGC2AC integrates nested graph constraints as application conditions into transformation rules.
    Each component can be used independently as an Eclipse-based tool.

Installation

  • 1. Install the Eclipse Modeling Tool, e.g., Eclipse Neon: https://www.eclipse.org/downloads/packages/release/Neon/3
  • 2. Install OCL in Eclipse: You can use the update manager in Eclipse to install OCL: E.g., in Eclipse Neon you use the link http://download.eclipse.org/releases/neon/ to install new plugins such as OCL SDK.
  • 3. Install Henshin: You can use the following update site to install Henshin using the update manager in Eclipse: http://download.eclipse.org/modeling/emft/henshin/updates/release
  • 4. Install OCL2AC: You can use the following update site to install our tool using the update manager in Eclipse: https://ocl2ac.github.io/home/release



Getting Started

  •   OCL2NGC


    To use OCL2NGC, two files are needed as inputs: an ecore file (.ecore) and an OCL file (.ecore.oclas):

    • (1) To get the OCL file (.ecore.oclas): Open the meta-model file (e.g., petrinet.ecore) using Open with->OCLinEcore Editor. Then, right mouse-click on the opened OCLinEcore and choose OCL->Save->Save Abstract Syntax.
    • (2) Select both files (.ecore) and (ecore.oclas), right mouse-click on them. Then choose OCL2AC-> Translate OCL Constraints to Nested Graph Constraints.
    • The output is a model file (.nestedconstraintmodel) which contains all the nested graph constraints being translated from OCL constraints.
    Note: Make sure that your domain (meta-model) is already registered in the Eclipse (i.e, the EMF model artifacts of your domain are generated and deployed in the Eclipse) before using the tool services.

  •   NGC2AC


    To use NGC2AC, two files are needed as inputs: a Henshin file (.henshin) and an nestedconstraintmodel file (.nestedconstraintmodel):

    • (1) Select both files (.henshin) and (.nestedconstraintmodel), right mouse-click on them. Then choose OCL2AC->Integrate Nested Graph Constraint in Rules as Application Condition. This opens the integrator wizard. The integrator wizard displays the constraints from the selected nestedconstraintmodel file and the rules from the selected Henshin file. Select one constraint and one rule.
    • The output is a new Henshin rule being contained in the Henshin file (.henshin).


  •   Graphical Formal View



      To generate and view the latex file which represents the graphical formal representation of a selected nested graph constraint or the left application condition of a selected Henshin rule:
    • For showing a nested graph constraint: Open the file (.nestedconstraintmodel) using the tree-based editor in Eclipse. Select a concrete constraint (the container node of it), right-mouse click on it to choose OCL2AC_NestedGraphConstraints.
    • For showing the application condition of a Henshin Rule: Open the Hesnhin file using the tree-based editor. Select a rule and right-mouse click on it to choose OCL2AC_Henshin.
    Note: Make sure that the Latex framework is already installed in your operation system and the latex path is registered in the environment variables. The time slot for rendering a latex file to pdf is set to 5 seconds for now.

  • Fig. Comparision of complexity of OCL and NGC for selected constraints

    Summarizing, both components of OCL2AC work fast. The translation of OCL to graph constraints is promising to increase the comprehensibility of con- straints, though a slight overhead in size is produced. Application conditions resulting from the integration process tend to be more complex then the original graph constraints. In most cases they are still easily understandable, but there are cases where the result becomes too complex to be understood anymore.
