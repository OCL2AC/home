OCL2AC: Automatic Translation of OCL Constraints to Graph Constraints and Application Conditions for Transformation Rules






OCL2AC Tool

   OCL2AC is a tool based on the Eclipse Modeling Framework (EMF) and consists of two main components:
    (1) OCL2NGC translates OCL constraints into nested graph constraints.
    (2) OCL2NGC integrates nested graph constraints as application conditions into transformation rules.
    Each component can be used independently as an Eclipse-based tool.

Installation

  • 1. Install the Eclipse Modeling Tool, e.g., Eclipse Neon: https://www.eclipse.org/downloads/packages/release/Neon/3
  • 2. Install OCL in Eclipse: You can use the update manager in Eclipse to install OCL: E.g., in Eclipse Neon you use the link http://download.eclipse.org/releases/neon/ to install new plugins such as OCL SDK.
  • 3. Install Henshin: You can use the following update site to install Henshin using the update manager in Eclipse: http://download.eclipse.org/modeling/emft/henshin/updates/release

  • Fig. Comparision of complexity of OCL and NGC for selected constraints

    Summarizing, both components of OCL2AC work fast. The translation of OCL to graph constraints is promising to increase the comprehensibility of con- straints, though a slight overhead in size is produced. Application conditions resulting from the integration process tend to be more complex then the original graph constraints. In most cases they are still easily understandable, but there are cases where the result becomes too complex to be understood anymore.
